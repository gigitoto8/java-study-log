## 本リポジトリの方針  
以下の条件を前提として制作しています。
- 資格取得で得た知識をテーマとする

- AIツール（ChatGPT）の提案を参考にしつつ、自力で構築する  

---

## アプリ概要  
CSV対応・学習ログ管理ツール（CUIベース）  
  
### 実装機能  
      - 学習記録の登録（CSVへ保存）  
        - 日付 (今回は学習優先のため、String型で対応)  
        - 科目（Java,SQL,C などを入力）  
        - 学習時間（分）  
        - メモ  
      - 一覧表示（CSVから読み込み）  
      - 検索機能  
      - 集計機能  
        - 科目ごとの合計学習時間  
### 技術制約  
        - Java標準APIのみ使用  
        - データベースは使わずCSVファイルで管理  
        - まずはCUIで実装（GUIやWeb化は後の学習で対応）  
### 設計方針  
        - クラス分割を意識する  
### ファイル構成  
  
        java-study-log/  
        ├ src/  
        │   ├ Main.java:実行  
        │   ├ StudyRecord.java:データ取扱い  
        │   ├ CsvService.java:CSVファイル操作  
        │   └ InputValidator.java:入力値チェック  
        └ data/  
            └ study_log.csv（自動生成）  

---

##  

### STEP1 : 基本構成とCSV保存機能の実装  
- 構成を作成し、各ファイルを実装  
- 学習記録をCSVに1件保存する機能を実装  
- この時点では、データはインスタンス生成時に登録  

---

### STEP2 : コンソール入力対応  
- Main.java : コンソール入力時、項目毎に入力を促す表示を追加
- CsvService.java : 日本語入力の文字化け対策として、CSV書き込み処理を修正  

---

### STEP3 : 一覧表示機能  
- Main.java : CSVデータの内容を表示する処理を追加  
- CsvService.java : `findAll`メソッド（CSV読み込み処理）を実装  
- StudyRecord.java : 表示用フォーマット処理を追加  

---

### STEP4 : 検索機能
- Main.java : 以下を追加  
  - 検索項目および検索ワードを入力し、該当するレコードを表示する処理を追加  
  - 項目の入力値が不正である場合の再入力を促す処理を追加  
  - 検索ワードに該当するものがない場合の表示を追加  
- CsvService.java : `findByCondition`メソッド（条件一致データの抽出処理）実装  
- StudyRecord.java : 検索処理で使用するgetterを追加  

---

### STEP5 : 集計機能  
- Main.java : `sumBySubject`メソッドで得られた集計結果を表示する処理を追加  
- CsvService.java : `sumBySubject`メソッド（科目毎の総学習時間を集計する機能）を実装  

---

### STEP6 : 入力バリデーション  
- 入力値チェック機能を有するInputValidator.javaをsrcディレクトリに新規作成  
- Main.java : InputValidatorクラスの`inputString`メソッドおよび`InputInt`メソッドに入力処理機能を移譲するため修正  

---

### STEP7：メニュー機能  
- アプリ起動直後にメニューを表示し、選択に応じて各機能を実行する機能  
- 処理実行後、メニューに戻るループ構造を実装（終了を選択するまで継続）
